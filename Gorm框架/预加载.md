# 预加载（Preload）

在我的理解中，预加载就是当你后续**还需要用**的某个用户在其他表上的关联信息时，那么在**第一次**查该用户在该表上的信息时就使用预加载技术**一起查出来**，来一劳永逸，节省性能。

当然，预加载技术会根据你这两张表**之间的关系**来生成**不同的SQL**来得到相关信息。

```go
// Preload Orders when find users
db.Preload("Orders").Find(&users)
// SELECT * FROM users;
// SELECT * FROM orders WHERE user_id IN (1,2,3,4);

db.Preload("Orders").Preload("Profile").Preload("Role").Find(&users)
// SELECT * FROM users;
// SELECT * FROM orders WHERE user_id IN (1,2,3,4); // has many 
// SELECT * FROM profiles WHERE user_id IN (1,2,3,4); // has one
// SELECT * FROM roles WHERE id IN (4,5,6); // belongs to

//带条件的预加载
db.Preload("Orders", "state NOT IN (?)", "cancelled").Find(&users)
// SELECT * FROM users;
// SELECT * FROM orders WHERE user_id IN (1,2,3,4) AND state NOT IN ('cancelled');
```



## Joins和Preload的区别

同样都是连表，Joins和Preload的区别在于Joins不会嵌套填充，只是普通的平面。

Preload能够嵌套填充。





## 预加载全部

```go
db.Preload(clause.Associations).Find(&users)
//gorm会自动解析并获得所有关联关系，预加载全部信息
```

GORM 会自动扫描 `User struct` 里所有“关联字段”，全部 Preload

比如：

```go
type User struct {
    Orders   []Order
    Company  Company
    Account  Account
}
```

 等价于：

```go
db.Preload("Orders").
   Preload("Company").
   Preload("Account")
```





## 嵌套预加载

```go
db.Preload("Orders.OrderItems.Product").Preload("CreditCard").Find(&users)
//SELECT * FROM users;
//SELECT * FROM orders WHERE user_id IN (1,2,3,...);
//SELECT * FROM order_items WHERE order_id IN (...);		相当于users -> Orders -> OrderItems -> Product
//SELECT * FROM products WHERE id IN (...);
//SELECT * FROM credit_cards WHERE user_id IN (...);
```
